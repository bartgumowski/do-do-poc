## 10.1 PWA Install Prompt

### Why first
Before building native apps, improve PWA installation rate.
iOS users who add to Home Screen get push notifications (iOS 16.4+).
Android users get a near-native experience already.

### Implementation in app.js
```js
let deferredInstallPrompt = null;

window.addEventListener('beforeinstallprompt', (e) => {
  e.preventDefault();
  deferredInstallPrompt = e;
  // Show custom install banner after user has been in app 30 seconds
  setTimeout(() => showInstallBanner(), 30000);
});

function showInstallBanner() {
  if (localStorage.getItem('install-dismissed')) return;
  // Show banner: "Add Do-Do to your home screen for push notifications and faster access"
  // [Add to Home Screen]  [Not now]
}
```

For iOS (where beforeinstallprompt doesn't fire):
Detect `navigator.standalone === false` on iOS Safari and show manual instructions:
"Tap [share icon] → Add to Home Screen"

## 10.2 Capacitor Wrapper

### What Capacitor does
Wraps the existing web app in a native iOS/Android container.
Zero code rewrite. The app runs inside a WKWebView on iOS, WebView on Android.
Adds access to native APIs: push (APNS/FCM), camera, file picker.

### Setup
```bash
npm install @capacitor/core @capacitor/cli @capacitor/ios @capacitor/android
npx cap init "Do-Do" "app.dodo.family" --web-dir "."
npx cap add ios
npx cap add android
```

### capacitor.config.json
```json
{
  "appId": "app.dodo.family",
  "appName": "Do-Do",
  "webDir": ".",
  "server": { "androidScheme": "https" },
  "plugins": {
    "PushNotifications": { "presentationOptions": ["badge", "sound", "alert"] }
  }
}
```

### Native push with Capacitor
Replace web push (VAPID) with Capacitor Push Notifications plugin:
```bash
npm install @capacitor/push-notifications
```

This uses APNS on iOS and FCM on Android - much more reliable than web push.
Requires Firebase project (for FCM) and Apple Push certificates.

## 10.3 App Store Submission

### iOS requirements
- Apple Developer account: $99/year at developer.apple.com
- App icons: 1024x1024 PNG (no alpha channel)
- Screenshots: iPhone 6.7" and 6.1" sizes required
- Privacy policy URL: https://do-do.app/legal
- App Review: usually 1-3 days, sometimes 1-2 weeks

### App Store metadata
- Name: "Do-Do - Family Coordination"
- Subtitle: "Co-parenting made simple"
- Category: Lifestyle (primary), Productivity (secondary)
- Keywords: co-parenting, family calendar, shared todo, custody schedule, parenting

### TestFlight first
Always submit to TestFlight before App Store.
Share TestFlight link with Bart and Art for real-device testing.

### Google Play requirements
- Google Play Developer account: $25 one-time at play.google.com
- App bundle: generated by `npx cap build android` (requires Android Studio)
- Privacy policy required
- Review: usually 1-3 days

## 10.4 Apple In-App Purchase

### The rule
If the app is distributed via the iOS App Store and users pay for a subscription,
Apple requires the purchase to go through Apple IAP (30% fee on first year, 15% on renewals).

### Options
1. **Web-only payment:** Don't offer subscription purchase inside the iOS app.
   User sees "Subscribe at do-do.app" - no in-app purchase button.
   Legal but poor UX. Apple may reject if subscription is prominently missing.

2. **Apple IAP + Stripe:** Support both.
   Detect platform. iOS app → Apple IAP via Capacitor In-App Purchases plugin.
   Web → Stripe. Backend reconciles both.

3. **Progressive Web App only:** Skip App Store entirely.
   PWA on Home Screen + web push. No App Store fees or rules.
   Totally valid for a B2C app targeting tech-aware parents.

### Recommendation
Launch as PWA first (no App Store fees or review delays).
Add App Store wrapper in v2 once product is stable and revenue justifies 30% cut.
